# 第二章：Transformer 革命

## 2.1 Transformer 之前

在 2017 年之前，序列建模意味着 RNN、LSTM 和 GRU——逐个处理 token。这种方式速度慢、难以并行化，并且在长距离依赖上表现不佳。

## 2.2 "注意力就是一切"（2017）

Vaswani 等人的论文提出了 Transformer 架构。核心创新：**注意力机制**让每个 token 可以直接"关注"所有其他 token，无论距离多远。

**关键组件：**
- **自注意力：** 每个 token 关注所有其他 token，按相关性加权
- **多头注意力：** 并行处理多个注意力模式
- **位置编码：** 为模型提供 token 顺序感（因为注意力是排列不变的）
- **编码器-解码器堆叠：** 原始架构；现代 LLM 使用仅解码器架构

## 2.3 缩放定律

Kaplan 等人（2020）和后来的 Chinchilla（2022）表明，模型性能与以下因素之间存在可预测的幂律关系：
- **参数量**（更大的模型）
- **数据量**（更多 token）
- **算力预算**（更多 FLOPs）

实际意义：向训练良好的 Transformer 投入更多算力可以预测性地提升性能。这一发现引发了缩放竞赛。

## 2.4 涌现能力

在特定的规模阈值上，模型突然展现出它们并未被显式训练的能力：
- 上下文学习（GPT-3）
- 思维链推理（PaLM、GPT-4）
- 指令遵循（InstructGPT/ChatGPT）
- 代码生成、翻译、工具使用

并非所有人都认同涌现是真实存在的——有些人认为这是测量假象——但无论如何，更大的模型在*感觉上*确实有质的不同。

## 2.5 后 Transformer 格局

Transformer 不再是唯一的选择：
- **Mamba、RWKV：** 具有线性时间推理的状态空间模型
- **混合架构：** Transformer 与 SSM 或 MoE 层结合
- **DeepSeek 的 MoE：** 用于高效扩展的混合专家模型

但核心洞见——基于注意力的架构随数据和算力扩展——仍然是后续一切的基础。
